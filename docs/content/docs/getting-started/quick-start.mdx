## Goal

We'll evolve a text similarity function that matches human judgments.
The baseline uses raw edit distance — here's what it gets wrong:

- "I love this movie" vs "I hate this movie" → **82% similar** (should be low)
- "the film was excellent" vs "the movie was great" → **45% similar** (should be high)
- "hi" vs "hello" → **20% similar** (should be high)

## 2. Configure the task

Populate the config with the system_message to give context about the problem, and which LLM to use.

```yaml title="config.yaml"
language: python
max_iterations: 50
checkpoint_interval: 5

llm:
  models:
    - name: "anthropic/claude-sonnet-4-6"
      weight: 1.0
  max_tokens: 8192

prompt:
  system_message: |-
    You are an expert in NLP and string similarity. Write a function
    that scores how alike two strings are, matching human intuition.
    Return a float between 0.0 (unrelated) and 1.0 (identical).
    Only use the Python standard library.

    The evaluation dataset includes typos, paraphrases, word
    reordering, negation, and unrelated pairs.

evaluator:
  timeout: 30

# Live dashboard — opens in your browser
monitor:
  enabled: true
```

## 3. Implement the evaluator

The evaluator runs in Docker. `evaluate.sh` is the entry point — it receives
a candidate program path and prints a JSON score to stdout. Here are all
four files:

  <Tab value="evaluator.py">
```python title="evaluator/evaluator.py"
#!/usr/bin/env python3
import importlib.util
import json
import sys

from scipy.stats import spearmanr

# 40 string pairs with human similarity ratings — see pairs.json
PAIRS = json.load(open("/benchmark/pairs.json"))


def main():
    program_path = sys.argv[1]

    # Load the candidate's similarity() function
    spec = importlib.util.spec_from_file_location("candidate", program_path)
    mod = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(mod)

    # Score every pair
    predicted = []
    for a, b, _ in PAIRS:
        try:
            score = max(0.0, min(1.0, float(mod.similarity(a, b))))
        except Exception:
            score = 0.0
        predicted.append(score)

    human = [h for _, _, h in PAIRS]
    correlation = spearmanr(predicted, human).statistic

    import random
    samples = random.sample(range(len(PAIRS)), 3)
    lines = [f"Spearman correlation: {correlation:.4f}", ""]
    for i in samples:
        a, b, h = PAIRS[i]
        lines.append(f"  '{a}' vs '{b}': predicted={predicted[i]:.2f}, human={h:.2f}")

    print(json.dumps({
        "status": "success",
        "combined_score": round(max(0.0, correlation), 4),
        "artifacts": {"feedback": "\n".join(lines)},
    }))


if __name__ == "__main__":
    main()
```

## 4. Provide a starting solution (optional)

If you skip this step, SkyDiscover inspects the evaluator and generates
a scaffold automatically. But providing one lets you control the starting
point — here we seed it with a basic edit distance implementation:

```python title="initial_program.py"
# EVOLVE-BLOCK-START
def similarity(a: str, b: str) -> float:
    """
    Return a similarity score between 0.0 (unrelated) and 1.0 (identical)
    for two input strings.

    This should capture not just character-level similarity but also
    meaning — paraphrases should score high, negations should score low,
    and typos should be forgiven.

    Only use the Python standard library (no external packages).
    """
    # Baseline: normalized Levenshtein distance
    if a == b:
        return 1.0
    if not a or not b:
        return 0.0

    m, n = len(a), len(b)
    dp = list(range(n + 1))
    for i in range(1, m + 1):
        prev = dp[0]
        dp[0] = i
        for j in range(1, n + 1):
            temp = dp[j]
            if a[i - 1] == b[j - 1]:
                dp[j] = prev
            else:
                dp[j] = 1 + min(dp[j], dp[j - 1], prev)
            prev = temp

    max_len = max(m, n)
    return 1.0 - dp[n] / max_len
# EVOLVE-BLOCK-END
```

Everything between `EVOLVE-BLOCK-START` and `EVOLVE-BLOCK-END` is what
SkyDiscover will mutate. Code outside the markers stays fixed.

## 5. Run the evolution

```bash
uv run skydiscover-run examples/text_similarity/initial_program.py \
  examples/text_similarity/evaluator/ \
  --config examples/text_similarity/config.yaml \
  --search best_of_n \
  --iterations 20
```

The live dashboard URL prints at startup — open it to watch solutions
evolve in real time.

In our test run, the correlation climbed from **0.26 to 0.69 in 5 iterations**
(~2 minutes):

| Iteration | Correlation | What changed |
|:---------:|:-----------:|:-------------|
| Baseline  | 0.26        | Raw edit distance |
| 1         | 0.57        | Token overlap + synonym matching |
| 3         | 0.69        | Negation penalty, tuned weights |

The winning solution combined synonym dictionaries, character n-gram overlap,
token Jaccard similarity, and negation detection — all discovered by the LLM
without any hand-tuning.

